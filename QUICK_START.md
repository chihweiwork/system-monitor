# 🚀 System Monitor - 快速開始指南

## 立即執行

```bash
# 方法 1: 開發模式
cargo run

# 方法 2: Release 模式（推薦，效能更好）
cargo run --release

# 或直接執行編譯好的二進位檔
./target/release/system-monitor
```

---

## ⌨️ 鍵盤快捷鍵

### 視圖切換（一鍵直達）

| 按鍵 | 視圖 | 顯示內容 |
|------|------|---------|
| **`1`** | CPU | CPU 使用率、User/System/Idle 時間 |
| **`2`** | Memory | 記憶體使用、Swap、快取 |
| **`3`** | Processes | 程序列表（可排序、過濾、查看詳情） |
| **`4`** | Network 🆕 | 網路流量（↓↑速度、封包統計） |
| **`5`** | Disk I/O 🆕 | 磁碟讀寫速度（MB/s） |
| **`6`** | Disk Usage 🆕 | 磁碟空間使用率 |
| **`7`** | GPU 🆕 | GPU 使用率、VRAM、溫度、功耗 |

### 循環切換

- **`Tab`** - 下一個視圖（1→2→3→4→5→6→7→1...）
- **`Shift+Tab`** - 上一個視圖（反向）

### 通用控制

- **`?`** 或 **`h`** - 顯示/隱藏說明畫面
- **`q`** 或 **`Esc`** - 退出程式

---

## 🎮 Process 視圖專用操作

在 Process 視圖（按 `3`）時可用：

### 導航
- `j` / `↓` - 向下移動
- `k` / `↑` - 向上移動  
- `g` / `Home` - 跳到第一個
- `G` / `End` - 跳到最後一個
- `PageUp` / `PageDown` - 翻頁

### 功能
- **`Enter`** - 查看程序詳細資訊
- **`/`** 或 **`f`** - 開啟過濾（按名稱搜尋）
- **`←`** / **`→`** - 切換排序欄位（PID/名稱/CPU/記憶體）
- **`s`** 或 **`Space`** - 切換升序/降序

## 🎨 顏色說明

### CPU & Memory
- 🟢 綠色：低使用率 (< 30%)
- 🟡 黃色：中等使用率 (30-70%)
- 🟠 橙色：高使用率 (70-85%)
- 🔴 紅色：非常高 (> 85%)

### Disk I/O
- 🟢 綠色：< 10 MB/s
- 🟡 黃色：10-100 MB/s
- 🔴 紅色：> 100 MB/s

### Disk Usage
- 🟢 綠色：< 70% 已用
- 🟡 黃色：70-90% 已用
- 🔴 紅色：≥ 90% 已用

### GPU Temperature
- 🟢 綠色：< 70°C
- 🟡 黃色：70-85°C
- 🔴 紅色：≥ 85°C

## 💡 提示與技巧

### 1. 快速找到佔用 CPU 的程序
```
1. 按 3 進入 Processes 視圖
2. 按 → 直到排序欄位為 "CPU%"
3. 按 s 確保是降序排列
4. 最上方就是最耗 CPU 的程序
```

### 2. 監控網路流量
```
1. 按 4 進入 Network 視圖
2. 觀察 ↓ (下載) 和 ↑ (上傳) 速度
3. 每秒自動更新
```

### 3. 檢查磁碟空間
```
1. 按 6 進入 Disk Usage 視圖
2. 查看各掛載點使用率
3. 紅色 (≥90%) 表示空間不足
```

### 4. GPU 監控（如果有 GPU）
```
1. 按 7 進入 GPU 視圖
2. 查看 GPU 使用率和 VRAM
3. 監控溫度避免過熱
```

## ⚙️ 系統需求

- **OS**: Linux (Ubuntu, Debian, Arch, etc.)
- **Rust**: 1.70+
- **Terminal**: 支援 ANSI 色彩的終端
- **最小尺寸**: 80x24 字元

### 可選硬體
- **GPU**: NVIDIA / AMD / Intel（sysfs 支援）
- **網路卡**: 任何 Linux 支援的網卡
- **磁碟**: 任何掛載的檔案系統

## 🐛 故障排除

### 程式無法啟動
```bash
# 確認編譯成功
cargo build --release

# 檢查錯誤訊息
cargo run 2>&1 | less
```

### GPU 視圖是空的
✅ **正常**！如果系統沒有 GPU 或驅動不支援，視圖會是空的。

### Network 視圖沒有資料
檢查網路連線：
```bash
# 確認有網路介面
ip link show

# 確認有流量
cat /proc/net/dev
```

### 效能問題
使用 release 模式：
```bash
cargo run --release
```
